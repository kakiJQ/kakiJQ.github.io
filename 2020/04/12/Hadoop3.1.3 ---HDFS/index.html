Hadoop3.1.3 —HDFS

一、概述

1. 定义
  • HDFS(Hadoop Distributed File System)是分布式文件系统
  • 通过目录树来定位文件
    • 与Linux 结构相同
  • 通过多台机器来联合存储数据
    • 各个节点的角色不同
  • 一次写入,多次读出,不适合修改
    • 可以在文件后面追加内容,但是不能修改
    • 如果修改, 只能先下载文件进行修改, 再上传
2. 优点
  • 高容错性
    • 多个副本
  • 适合处理大数据
  • 可以在廉价机上,通过多个副本,提高可靠性
3. 缺点
  • 不适合低延迟数据访问
    • 毫秒做不到
  • 无法高效对小文件进行储存
    • 没个文件的元数据(namenode)大小是差不多的
    • 可能寻找的时间大于读取(或MapReduce计算)的时间
    • 整个hadoop是怕小文件的
  • 不支持并发写入, 不支持文件随机修改
    • 不支持多线程
    • 只能对文件进行追加
4. HDFS组成架构
  • NameNode: Master
    • 管理HDFS名称空间 (目录结构)
    • 配置副本策略 (存几个副本, 哪个节点)
    • 管理块(block)映射信息
    • 处理客户端读写请求 (读写都先经过NameNode)
  • DataNode: Slaver
    • 存储实际的数据块
      • 客户端发送请求
      • NameNode进行分配给dn
      • DateNode接收nn命令并操作
    • 执行数据块的读写操作
  • Secondary NameNode:
    • 只能修复NameNode
    • 不能替换NameNode
  • Client
    • 文件切分, 文件上传HDFS时, Client将文件切分成一个一个block(默认128m)进行上传
    • 与NameNode交互, 获取文件位置信息
    • 与DataNode交互, 读取或写入数据
    • Client提供一些命令来管理HDFS, 比如NameNode初始化
    • Client可以通过命令来访问HDFS, 比如对HDFS增删查
5. HDFS文件块大小

  • HDFS的文件在物理上是分块存储(block), 块的大小是可以通过配置参数(dfs.blokcsize)来决定

  • 由client先切块上传到hdfs的NameNode上

  • 所有的块信息在NameNode上

  • 由NameNode来找到数据在哪个DataNode

  • 满足块大小(默认128m), 块就多大

  • 不满足块大小(默认128m), 会分配另一个块, 前一块则为128m

  • 其块大小与硬件性能有关, 寻址为传输时间的1%, 则为最佳状态
